1. Why is object-oriented development so important in programming?

    2. Because it allows for better organization of code, making it more modular and reusable. It also helps in modeling real-world entities, making it easier to understand and maintain the code.

  2. In Task 1, we create a static object called "AccountType." Explain what this object is used for and how this type of object can help us in software development.

    3. This object is used to define different types of bank accounts in our application. By using a static object for this purpose, we can easily manage and reference the various account types throughout our code. This approach promotes consistency and reduces the likelihood of errors, as we can rely on a single source of truth for account type definitions. Prevents hardcoding

  3. Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function.

    4. We want to "hide" the internal implementation details of a class to protect its state and ensure that it can only be modified in controlled ways. Using # or closures in constructors for creating private members. Public members then we define normally in the class or return them from the constructor.

  4. Explain what happens when we use the "new" command in JavaScript.

    5. "new" command creates a new empty object. This object is then linked to the prototype of the constructor function that is being called. The constructor function is executed with the new object as its context, and if the constructor function does not explicitly return an object, the new object is returned by default.
